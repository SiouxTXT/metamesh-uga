## 1. Visione e Architettura Generale

### 1.1 Missione
Fornire un **comando unico** (`metamesh connect`) che dia accesso immediato a **tutti gli MCP server esistenti**, senza configurazione, senza installazioni manuali, senza gestione delle dipendenze.

### 1.2 Filosofia Progettuale
- **Serverless**: Zero infrastruttura da gestire
- **Stateless**: Ogni richiesta è indipendente
- **Open Source**: Core libero, servizi a pagamento
- **Costo Zero**: Sfruttare al massimo i free tier
- **Edge Native**: Esecuzione su Cloudflare Workers (300+ data center)

### 3.1 Livello 1 - Esposto (Public Gateway)

#### 3.1.1 Worker: `metamesh-gateway`

**Endpoint**:
```yaml
REST:
  - POST /v1/call              # Chiamata tool MCP
  - GET  /v1/tools             # Lista tool disponibili
  - GET  /v1/tools/:name       # Dettaglio tool
  - POST /v1/auth/login        # Genera JWT
  - GET  /v1/usage             # Statistiche utente (pro)

MCP:
  - JSON-RPC su SSE            # Per AI Assistant
  - /sse                       # EventSource endpoint
  - /message                   # JSON-RPC endpoint
```

**Rate Limiting**:
- Free: 100 richieste/minuto
- Pro: 1.000 richieste/minuto
- Enterprise: 10.000 richieste/minuto

**Routing Logic**:
```javascript
async function routeRequest(toolName, params) {
  // Step 1: Check cache (KV)
  const cached = await KV.get(`tool:${toolName}`);
  if (cached) return executeWASM(cached, params);
  
  // Step 2: Query D1 per categoria
  const tool = await D1.prepare(
    "SELECT * FROM tools WHERE name = ?"
  ).bind(toolName).first();
  
  // Step 3: Direct L1 → L5 (bypass aggregatori)
  const wasm = await R2.get(`wasm/${toolName}.wasm`);
  return executeWASM(wasm, params);
}
```

#### 3.1.2 Autenticazione

**Modalità Supportate**:
1. **API Key**: Header `X-API-Key: sk_...`
2. **JWT**: Header `Authorization: Bearer <jwt>`
3. **OAuth 2.0**: (Enterprise)

**Flusso JWT**:
```javascript
async function generateJWT(userId, plan) {
  return jwt.sign(
    { sub: userId, plan, exp: Math.floor(Date.now() / 1000) + 3600 },
    SECRET_KEY,
    { algorithm: 'HS256' }
  );
}
```

### 3.2 Livello 2 - Motore di Ricerca (Non Esposto)

**Worker**: `metamesh-discovery`

**Funzione**:
- Cron trigger ogni 6 ore
- Interroga Registry MCP ufficiale
- Aggiorna catalogo su D1

**Codice**:
```javascript
async function discoverNewMCPs() {
  const response = await fetch(
    'https://registry.mcp.io/v0.1/servers'
  );
  const servers = await response.json();
  
  for (const server of servers) {
    // Verifica se esiste già
    const existing = await D1.prepare(
      "SELECT * FROM tools WHERE name = ?"
    ).bind(server.name).first();
    
    if (!existing) {
      await D1.prepare(
        "INSERT INTO tools (name, version, category, schema, source_url) VALUES (?, ?, ?, ?, ?)"
      ).bind(
        server.name,
        server.version,
        server.category,
        JSON.stringify(server.schema),
        server.source_url
      ).run();
    }
  }
}
```

### 3.3 Livello 3 - Aggregatore di Aggregatori (Non Esposto)

**Worker**: `metamesh-aggregator`

**Funzione**:
- Coordina le richieste tra categorie
- Gestisce fallback se un aggregatore di categoria fallisce
- Mantiene il routing table L1 → L4/L5

**Routing Table** (su D1):
```sql
CREATE TABLE routing (
  tool_name TEXT PRIMARY KEY,
  category TEXT,
  worker_url TEXT,  -- URL del Worker di categoria
  latency_ms INTEGER,
  last_used TIMESTAMP
);
```

**Fallback Logic**:
```javascript
async function routeToCategory(toolName) {
  const routing = await D1.prepare(
    "SELECT * FROM routing WHERE tool_name = ?"
  ).bind(toolName).first();
  
  // Tentativo 1: Worker di categoria
  try {
    return await fetch(routing.worker_url, { ... });
  } catch (err) {
    // Fallback: Chiamata diretta L1 → L5
    return await directWASMExecution(toolName);
  }
}
```

### 3.4 Livello 4 - Aggregatori di Categoria (Non Esposti)

**Worker per categoria**: `metamesh-category-{a,b,c}`

**Categorie Iniziali**:
| Categoria | Worker | Esempi MCP |
|-----------|--------|------------|
| Communication | `cat-comm` | Gmail, Slack, Telegram |
| Development | `cat-dev` | GitHub, GitLab, Jira |
| Data | `cat-data` | PostgreSQL, Snowflake, BigQuery |
| AI/ML | `cat-ai` | OpenAI, Anthropic, Gemini |

| Infrastructure | `cat-infra` | AWS, GCP, Azure, Kubernetes |
| Finance | `cat-fin` | Stripe, Plaid, Coinbase |

**Funzione per Worker**:
- Mantiene una cache locale dei tool della sua categoria
- Esegue chiamate ai singoli MCP (Livello 5)
- Aggrega risultati se multipli tool nella stessa categoria

**Codice Template**:
```javascript
export default {
  async fetch(request, env) {
    const { toolName, params } = await request.json();
    
    // Cache check (KV locale)
    const cached = await env.CATEGORY_CACHE.get(toolName);
    if (cached) return new Response(cached);
    
    // Esecuzione tool
    const wasm = await env.R2.get(`wasm/${toolName}.wasm`);
    const result = await executeWASM(wasm, params);
    
    // Cache result per 5 minuti
    await env.CATEGORY_CACHE.put(toolName, JSON.stringify(result), { expirationTtl: 300 });
    
    return new Response(JSON.stringify(result));
  }
};
```

## 4. Motori di Gestione

### 4.1 Motore di Inserimento

**Worker**: `metamesh-inserter`

**Trigger**: Ogni 6 ore

**Pipeline**:
1. **Scoperta**: Legge nuovo MCP da Registry
2. **Download**: Scarica sorgente (GitHub, npm, PyPI)
3. **Compilazione**: Compila in WASM
   - Python → Pyodide
   - JavaScript → wasm-pack
   - Rust → wasm-pack
   - Go → TinyGo
4. **Test**: Esegue test di compatibilità
5. **Storage**: Carica su R2
6. **Indexing**: Aggiorna D1 e routing table

**Codice Compilazione**:
```javascript
async function compileToWASM(source, language) {
  switch(language) {
    case 'python':
      // Usa Pyodide per compilare Python → WASM
      return await pyodide.runPythonAsync(source);
    case 'javascript':
      // Usa wasm-pack per compilare JS → WASM
      return await wasmPack.compile(source);
    default:
      throw new Error(`Unsupported language: ${language}`);
  }
}
```

### 4.2 Motore di Aggiornamento

**Worker**: `metamesh-updater`

**Trigger**: Ogni 12 ore

**Logica**:
1. Query Registry per versioni aggiornate
2. Confronta con D1
3. Se nuova versione → compila in WASM
4. Carica su R2 con versione (es. `gmail_v1.2.wasm`)
5. Mantiene versione precedente per 30 giorni (fallback)
6. Aggiorna D1 con nuova versione

**Fallback Automatico**:
```javascript
async function executeWithFallback(toolName, params) {
  // Tentativo con ultima versione
  const latest = await D1.prepare(
    "SELECT version FROM tools WHERE name = ? ORDER BY version DESC LIMIT 1"
  ).bind(toolName).first();
  
  try {
    return await executeWASM(toolName, latest.version, params);
  } catch (err) {
    // Fallback a versione precedente
    const previous = await D1.prepare(
      "SELECT version FROM tools WHERE name = ? AND version < ? ORDER BY version DESC LIMIT 1"
    ).bind(toolName, latest.version).first();
    
    return await executeWASM(toolName, previous.version, params);
  }
}
```

### 4.3 Motore di Eliminazione

**Worker**: `metamesh-eliminatore`

**Trigger**: Ogni 24 ore

**Criteri di Eliminazione**:
1. **Deprecato da > 90 giorni**: Flag `deprecated = TRUE`
2. **Nessun utilizzo da > 180 giorni**: Flag `inactive = TRUE`
3. **Versione obsoleta**: Rimosse dopo 30 giorni

**Pipeline**:
```javascript
async function cleanupDeprecated() {
  // 1. Trova tool deprecated
  const deprecated = await D1.prepare(
    "SELECT * FROM tools WHERE deprecated = TRUE AND deprecated_since < datetime('now', '-90 days')"
  ).all();
  
  for (const tool of deprecated) {
    // 2. Rimuovi da R2
    await R2.delete(`wasm/${tool.name}_${tool.version}.wasm`);
    
    // 3. Rimuovi da D1
    await D1.prepare(
      "DELETE FROM tools WHERE id = ?"
    ).bind(tool.id).run();
    
    // 4. Notifica utenti che lo usano
    await notifyUsers(tool.name, tool.version);
  }
}
```

## 5. Esecuzione e Runtime WASM

### 5.1 Runtime WASM su Cloudflare Worker

**Strumento**: `wazero` (Go) compilato in WASM e integrato nel Worker

**Limitazioni**:
- Memoria: 128 MB per istanza
- CPU Time: 50 ms per esecuzione (hard limit)
- File System: Virtuale, non persistente
- Networking: Limitato (solo HTTP/HTTPS)

**Implementazione**:
```javascript
import { Wazero } from 'wazero';

export default {
  async fetch(request, env) {
    const { toolName, params } = await request.json();
    
    // Carica WASM da R2
    const wasmBuffer = await env.R2.get(`wasm/${toolName}.wasm`);
    
    // Istanzia runtime
    const runtime = new Wazero();
    const instance = await runtime.instantiate(wasmBuffer);
    
    // Esegui con timeout
    const result = await Promise.race([
      instance.exports.callTool(params),
      new Promise((_, reject) => 
        setTimeout(() => reject(new Error('Timeout 5s')), 5000)
      )
    ]);
    
    return new Response(JSON.stringify(result));
  }
};
```

### 5.2 Cold Start Mitigation

**Strategie**:
1. **Pre-warming**: Cron trigger che ogni 6 ore esegue i top 100 tool
2. **Cache L2**: `caches.default` per WASM dopo primo download
3. **Progress Indicator**: Messaggio *"Primo caricamento del tool, attendi..."*

**Codice Pre-warming**:
```javascript
async function warmCache() {
  // Top 100 tool per popularity_score
  const topTools = await D1.prepare(
    "SELECT name FROM tools ORDER BY popularity_score DESC LIMIT 100"
  ).all();
  
  for (const tool of topTools) {
    // Carica in cache
    const wasm = await R2.get(`wasm/${tool.name}.wasm`);
    await caches.default.put(
      `wasm:${tool.name}`,
      new Response(wasm)
    );
  }
}
```

### 5.3 Gestione degli Errori

**Categorie di Errore**:
1. **Tool non trovato**: 404
2. **Timeout**: 408
3. **Parametri invalidi**: 400
4. **MCP a valle fallisce**: 502
5. **Rate limit superato**: 429

**Risposta Standard**:
```json
{
  "error": {
    "code": "TOOL_TIMEOUT",
    "message": "Il tool ha superato il timeout di 5 secondi",
    "tool": "gmail_send_email",
    "timestamp": "2026-06-16T12:00:00Z"
  }
}
```

## 8. Sicurezza e Autenticazione

### 8.1 Gestione delle Chiavi API

**Generazione**:
```javascript
function generateAPIKey() {
  return `sk_${crypto.randomUUID().replace(/-/g, '')}`;
}
```

**Validazione**:
```javascript
async function validateAPIKey(apiKey) {
  const user = await D1.prepare(
    "SELECT * FROM users WHERE api_key = ?"
  ).bind(apiKey).first();
  
  if (!user) throw new Error('Invalid API key');
  if (user.plan === 'free' && user.usage_count > 1000) {
    throw new Error('Monthly limit exceeded');
  }
  
  return user;
}
```

### 8.2 Cifratura delle Configurazioni

**Algoritmo**: AES-256-GCM

**Implementazione**:
```javascript
async function encryptConfig(data, userId) {
  const key = await deriveKey(userId);  // Deriva da chiave utente
  const iv = crypto.getRandomValues(new Uint8Array(12));
  const encrypted = await crypto.subtle.encrypt(
    { name: 'AES-GCM', iv },
    key,
    new TextEncoder().encode(JSON.stringify(data))
  );
  
  return {
    iv: Buffer.from(iv).toString('base64'),
    data: Buffer.from(encrypted).toString('base64')
  };
}
```

### 8.3 Rate Limiting

**Implementazione**:
```javascript
async function checkRateLimit(userId, plan) {
  const limit = plan === 'free' ? 100 : plan === 'pro' ? 1000 : 10000;
  const key = `ratelimit:${userId}`;
  
  const current = await KV.get(key);
  const count = current ? parseInt(current) : 0;
  
  if (count >= limit) {
    throw new Error(`Rate limit exceeded (${limit}/min)`);
  }
  
  await KV.put(key, count + 1, { expirationTtl: 60 });
}
```

- Rate limit superato > 100 volte

- Telegram (gratuito)
- Email (via Cloudflare Email Routing)
- Discord (webhook)

**Codice**:
```javascript
async function checkAlerts() {
  const stats = await ANALYTICS.sql(`
    SELECT 
      COUNT(*) as total,
      COUNTIF(status = 'error') as errors,
      AVG(latency_ms) as avg_latency
    FROM metamesh_usage
    WHERE timestamp > NOW() - INTERVAL 10 MINUTE
  `);
  
  const errorRate = stats.errors / stats.total;
  
  if (errorRate > 0.05) {
    await sendAlert(`⚠️ Error rate high: ${(errorRate * 100).toFixed(2)}%`);
  }
  
  if (stats.avg_latency > 500) {
    await sendAlert(`⚠️ High latency: ${stats.avg_latency}ms`);
  }
}
```

export async function handleMCP(request, env) {
  const { method, params, id } = await request.json();
  
  if (method === 'tools/list') {
    const tools = await env.DB.prepare(
      "SELECT name, description, schema FROM tools WHERE deprecated = FALSE"
    ).all();
    return JSONRPCResponse(id, tools);
  }
  
  if (method === 'tools/call') {
    const result = await executeTool(params.name, params.arguments, env);
    return JSONRPCResponse(id, result);
  }
  
  return JSONRPCError(id, 'Method not found');
}
```

### 12.2 Discovery (Livello 2)

export default {
  async scheduled(event, env, ctx) {
    await discoverNewMCPs(env);
  }
};

async function discoverNewMCPs(env) {
  const response = await fetch('https://registry.mcp.io/v0.1/servers');
  const servers = await response.json();
  
  for (const server of servers) {
    const existing = await env.DB.prepare(
      "SELECT id FROM tools WHERE name = ?"
    ).bind(server.name).first();
    
    if (!existing) {
      await env.DB.prepare(
        `INSERT INTO tools (name, version, category, description, schema, source_url)
         VALUES (?, ?, ?, ?, ?, ?)`
      ).bind(
        server.name,
        server.version,
        server.category,
        server.description,
        JSON.stringify(server.schema),
        server.source_url
      ).run();
    }
  }
}
```

export async function onRequest(context) {
  const { env } = context;
  
  const result = await env.ANALYTICS.sql(`
    SELECT 
      DATE(timestamp) as date,
      COUNT(*) as requests,
      COUNTIF(status = 'error') as errors,
      AVG(latency_ms) as avg_latency
    FROM metamesh_usage
    WHERE timestamp > NOW() - INTERVAL 30 DAY
    GROUP BY date
    ORDER BY date
  `);
  
  return new Response(JSON.stringify(await result.json()), {
    headers: { 'Content-Type': 'application/json' }
  });
}
